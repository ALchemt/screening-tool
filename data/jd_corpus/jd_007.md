# LLM Application Developer @ SupportOS

## About SupportOS

SupportOS builds AI-powered customer support automation for SaaS companies, reducing ticket volume by 40% through intelligent triage, auto-responses, and suggested replies. We serve 120+ B2B SaaS customers including several YC companies.

## Role

We're hiring a junior LLM Application Developer to build and deploy RAG-powered support agents that answer customer questions using help docs, past tickets, and product data. You'll work with FastAPI to create production APIs, write and refine prompts for GPT-4 and Claude 3.5, and implement basic evaluation pipelines to measure answer quality. You'll deploy your work to Google Cloud Run and collaborate with our product team to iterate based on customer feedback.

## Responsibilities

- Build FastAPI endpoints for LLM-powered support features (ticket triage, auto-responses, suggested replies)
- Implement RAG pipelines using LangChain with vector search over help docs and historical tickets
- Write, test, and version control prompt templates for consistent LLM output
- Develop basic evaluation scripts to measure answer accuracy, relevance, and hallucination rates
- Deploy applications to Google Cloud Run with CI/CD via GitHub Actions
- Monitor API performance, latency, and error rates using Google Cloud Logging and Sentry
- Collaborate with support ops team to gather feedback and improve AI response quality

## Required Qualifications

- 1-2 years experience with Python and building REST APIs (FastAPI, Flask, or Django)
- Hands-on experience calling OpenAI or Anthropic APIs and handling responses
- Familiarity with prompt engineering and few-shot learning techniques
- Basic understanding of RAG concepts and vector databases (Pinecone, Qdrant, ChromaDB)
- Experience deploying Python applications to cloud platforms (GCP, AWS, or Azure)
- Comfortable with Git, Docker, and CI/CD pipelines
- Strong debugging skills and attention to detail

## Nice-to-Have

- Experience with customer support platforms (Zendesk, Intercom, Front)
- Familiarity with LangChain or LlamaIndex for RAG pipelines
- Knowledge of evaluation frameworks (RAGAS, LangSmith, or custom)
- Understanding of streaming responses and WebSocket APIs
- Prior experience in a customer-facing or support role
